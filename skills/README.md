# Agent Skills

This repository keeps reusable tool guidance in plain repo-local markdown under
`skills/`.

Layout conventions:

- one subdirectory per tool or skill family
- one primary `SKILL.md` file per skill
- optional supporting notes can live beside `SKILL.md` if a future skill needs them

Current skills:

- `skills/github-cli/SKILL.md`
- `skills/duckdb/SKILL.md`

Discovery expectations:

- Codex can read these files directly from the repository.
- Claude can reference the same markdown files as repo-local instructions.
- GitHub Copilot can reference the same markdown files from the workspace.

These skills are intentionally maintained as plain markdown assets. They are not
generated by `purser init`.
